## M06 - Resume Review (AI Similarity + LLM Summary)
Requirements: FRCD-R01, FRCD14-FRCD22, FRP-A02, FRP-A03
- 3-panel layout: Left=Waiting list (badges, Dept filter, Include Starred); Center=Resume+docs; Right-top=Applicant Info+LLM Summary; Right-bottom=Position panel (Overall Score ranked)
- Overall Score = weighted avg of JD similarity + Hard Skill similarity using Weight Popup values
- Pre-compute similarity for Next Applicant on load
- HR marks up to 3 suitable positions (position_marks - reference only, no new Application)
- Buttons: Interest (Pass->In Progress) / Reject (Drop, auto-send SYS02) / Star (->Talent Pool, auto-Reject, NO rejection Email) / End Review / Next
- When new Job published, auto-compute similarity for all Starred Candidates
- If OCR failed, show Retry OCR button instead of LLM Summary

## M07 - Action Pipeline - Test (multi-round)
Requirements: FRP-A04, FRP-A04b-FRP-A04f, FRP-A08, FRP-A09, FRAT01-FRAT09, FRST01-FRST09
- HR initiates Action Test after Review; badge per Applicant shows round number
- HR selects Applicants; first round uses default test linked to Job; subsequent HR chooses from library
- Test modes: Online (email link, re-openable until Submit or expiry, auto-save draft) or Onsite
- General Test (HR): MCQ auto-graded; Essay/Short Answer graded by HR
- Technical Test (Supervisor): MCQ auto-graded; Essay/Short Answer graded by creator Supervisor
- Anti-cheat: no copy/paste, tab-switch detection; Timer per test or per question; random order
- Grading Inbox for Supervisor: badge count, grade per question, Submit sends FRNOT05 to HR
- Phase 1 Grading: HR sees scores real-time; waiting-for-grading until all manual questions graded
- Phase 2 Selection: leaderboard by score (live); decisions: Test again / Interview / Hire / Drop (NO Transfer from Test)
- HR/Supervisor CRUD test library; link default test to Job creation

## M08 - Action Pipeline - Interview (multi-round, Scoring Criteria, Invite Links)
Requirements: FRP-A05, FRP-A05b-FRP-A05g, FRSC-I01-FRSC-I08, FRH19-FRH36, FRP-A06, FRP-A07
- HR initiates Action Interview; badge per Applicant shows round number
- HR selects Applicants; HR defines Scoring Criteria (criteria name + max score per item)
- Supervisor can create Criteria only for rounds they are assigned to
- HR assigns interviewers: Supervisor (regular) and/or Temp Interviewers (invite link, 256-bit token)
- Auto-create temp account on first click; scoped to assigned Applicants+criteria; auto-deactivated after round or expiry; HR can revoke
- Phase 1 Grading: grade per criteria + note real-time; HR sees per-interviewer scores; shows who has not submitted
- Phase 2 Selection: leaderboard avg-of-averages; per-interviewer comparison; decisions final once made
- Next Action in Phase 2: Test / Interview / Transfer / Hire / Drop
- HR sends interview invitation Email using TPL05
- HR views and revokes invite links list

## M09 - Cross-Position Transfer
Requirements: FRCD-T01-FRCD-T08, FRP-A06
- Transfer ONLY during/after Interview (Phase 1 or Phase 2) - NOT after Test
- On Transfer: old Application closes + slot returned; new Application in target Position; history carries over
- Transfer unlimited times (A->B->C); Applicant unaware until Offer issued
- If target Position Closed: system shows warning before confirm
- Block sending Test from wrong Position

## M10 - Talent Pool / Star Feature
Requirements: FRCD20, FRCD22, FRP-A03
- Note: Star button behavior belongs to M06; M10 covers auto-compute + Starred display only
- Starred Candidates auto-computed against all new Jobs when published
- Appear in Resume Review Waiting list with star badge; Include Starred checkbox filter
- Starred Candidates removed from pool if Blacklisted

## M11 - Candidate Management (List, Detail, OCR, Blacklist)
Requirements: FRCD04-FRCD13, FRCD23-FRCD27
- Candidate List: name, email, app count, latest status; search name/email; filter status/position
- Candidate Detail: personal info (HR can edit name); all Application history; docs + Test/Interview scores; download docs; Timeline with Transfer history
- OCR auto-trigger on Resume after submit (PDF/DOCX/image -> text+vector); if fails show Retry OCR; non-blocking
- Blacklist: mandatory reason; block future apps; hide from Waiting list; remove from Starred pool
- Unblacklist: mandatory reason; restore normal; history preserved
- Blacklist History Log: type, reason, timestamp, HR name - shown as timeline in Candidate Detail

## M12 - Supervisor Requisition
Requirements: FRS03-FRS09
- Supervisor creates Requisition: title, dept, headcount, JD, Hard Skills, education, experience, job type
- Submit to HR: status->Pending; HR notified via FRNOT01
- Supervisor tracks status: Draft / Pending / In-Progress / Published
- HR views and edits Requisition before creating Job Posting
- HR converts Requisition to Job Posting 1-click; Job form pre-filled from Requisition data
- HR notifies Supervisor when Job published via FRNOT02

## M13 - Notification System
Requirements: FRNOT01-FRNOT10, FRH44-FRH47
- In-app Notification bell/inbox in main nav for HR and Supervisor
- 10 defined events: new Requisition / Job published / new Applicant / all tests submitted / Tech test graded / all interview scores / assigned to interview / assigned to grade Tech Test / Offer response / Job auto-closed
- Per-user toggle per channel (In-App vs Email) independently, saved per user
- Unread badge count on bell; In-App accumulates until read
- Default: FRNOT03+FRNOT09+FRNOT10=In-App only; all others=In-App+Email
- Disabling Email does not affect In-App and vice versa

## M14 - Setup / Pre-configuration
Requirements: FRSC01-FRSC05, FRSC10-FRSC19
NOTE: FRSC06-FRSC09 (Form Builder) already done in M05 - SKIP.
- Email Templates: SYS hardcoded (SYS01/SYS02/SYS03); TPL HR-managed (TPL04/TPL05/TPL06/TPL07 must exist before action); system blocks if required TPL missing
- Job Detail Template: HR creates standard JD template; default when creating Job
- Test Library: HR manages General Test library; Supervisor manages Technical Test library (CRUD with ref check)
- Manage Data: HR CRUD Department (ref check before delete), Employment Type, Experience Level, Hard Skill; Filter+Refresh
- JD+Hard Skill Embedding: after Create/Edit Job, send to AI-Service for text embedding+vector storage; store weight values

Requirements: FRI18N01-FRI18N07, FRTHM01-FRTHM06
- I18N via next-intl: Thai/English toggle in Header for both apps; default=Thai; persist in user profile or localStorage

- Persist theme in user profile or localStorage
